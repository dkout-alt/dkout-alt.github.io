MySQL 数据模型
关系型数据库：由多张能互相连接的二维表组成的数据库
优点：使用表结构，格式一致，便于维护、通用sql语言操作、数据存放在磁盘中 安全

简介，通用语法，分类，DDL DML DQL DCL 中间那几个缩写是 定义，操作Manipulation，查询，控制

RDBMS 指的是关系型数据库管理系统
SQL 对大小写不敏感。
由 SQL 查询程序获得的结果被存放在一个结果集中。使用编程函数在结果集中进行导航。
    SELECT DISTINCT * FROM TABLE_NAME;
其他的通配符
    [charlist] 否定为 [!charlist]
    e.g. select * from XXX where City LIKE '[ALN]%' -- 筛选出以 A、L、N 开头的人 

IN 操作符 允许在 WHERE 子句中规定多个值
    e.g. SELECT * FROM Persons WHERE LastName IN ('Adams','Carter') -- 选择姓氏为 Adam 或 Carters 的人。

SQL Join: 从多个表获取结果
    INNER JOIN 在表中存在至少一个匹配时，该关键字返回行，如
        SELECT COLUMN_NAME FROM TABLE_1
        INNER JOIN TABLE_2
        ON table1.columnname=table2.columnname;
    LEFT JOIN 会从左表中返回所有的行，即使右表中没有匹配的行。“穷人”
    RIGHT JOIN 相反，"赔钱货"
    FULL JOIN 只要其中某个表存在匹配，就返回行，如
        现在，我们希望列出所有的人，以及他们的定单，以及所有的定单，以及定购它们的人。
        SELECT Persons.LastName, Persons.FirstName, Orders.OrderNo
        FROM Persons
        FULL JOIN Orders
        ON Persons.Id_P=Orders.Id_P
        ORDER BY Persons.LastName

UNION 用来合并两个或多个 SELECT 语句的结果集，两个结果集列数量必须相同，数据类型必须相似， 每条S语句中的列的顺序必须相同
    它默认选取不同的值，UNION ALL 将允许重复的值,如 列出中美的所有雇员
        SELECT E_Name FROM Employees_China
        UNION ALL
        SELECT E_Name FROM Employees_US

SELECT INTO 语句从一个表中选取数据，然后把数据插入另一个表中。
    SELECT column_name(s)
    INTO new_table_name [IN externaldatabase] 
    FROM old_tablename

添加 UniQUE 约束 
    ALTER TABLE Persons
    ADD UNIQUE (Id_P)
撤销UNIQUE约束
    ALTER TABLE Persons
    DROP INDEX uc_PersonID

主键约束也类似 但如果使用 ALTER TABLE 语句添加主键，必须把主键列声明为不包含 NULL 值（在表首次创建时）。
DROP PRIMARY KEY 用来撤销主键约束

外键约束
    FOREIGN KEY 指向另一个表中的 PRIMARY KEY ，语法
        CREATE TABLE ORDERS(
        Id_P int FOREIGN KEY REFERENCES Persons(Id_P)
        );

    FOREIGN KEY 约束用于预防破坏表之间连接的动作。
    FOREIGN KEY 约束也能防止非法数据插入外键列，因为它必须是它指向的那个表中的值之一。

SQL CHECK 约束
    创建时：
        Id_P int NOT NULL CHECK (Id_P>0),
    修改时：
        ALTER TABLE Persons
        ADD CHECK (Id_P>0)
    命名：
        在 Check 前加上 Constraint chk_person 即可
    撤销：
       DROP CHECK chk_Person

创建索引:高效查询，用户不可见
    CREATE INDEX index_name
    ON table_name (column_name)

撤销索引
    ALTER TABLE NAME
    DROP INDEX INDEX_NAME

删除表中内容 不删除表本身 TRUNCATE TABLE
AUTO-INCREMENT 关键字能够在每次插入新纪录时，自动创建主键字段的值

    视图是可视化的表,语法为
