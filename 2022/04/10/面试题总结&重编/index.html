面试题总结&重编：容器（集合）

我们采用新的叙述方式：一切从问题出发，最终回归于定义（源码）

问题：首先，为什么要使用集合？

  • 假设情况：我需要存储一组类型相同（Optional）的数据
    • 使用容器 “数组”
      • 数组的长度声明后不可变，怎么处理？
      • 如果类型不再相同，数组也无法储存。
      • 数组存储的是有序，可重复的对象。如果要存储的对象并非有序可重复呢？
    • 使用集合，能存储不同类型，不同数量的对象，可以保存具有映射关系的数据

在决定使用集合后，了解 Java 中 Collection 框架

问题：说说 List, Set, Queue, Map 的区别

“是什么” 与 底层数据结构

  • List, Set, Queue 都继承于 Collection 且实现了 java.lang.Iterable 接口

    • List 有序，可重复 <!– HyperLink to “深入了解 List” >
      • ArrayList Object[] 数组
      • Vector Object[] 数组
      • LinkedList 双向链表
    • Set 无序，不可重复
      • HashSet 基于 HashMap 实现的；无序，唯一
      • LinkedHashSet 是 HashSet 的子类，内部通过 LinkedHashMap 实现
      • TreeSet 红黑树（自平衡的排序二叉树）；有序，唯一
        • 红黑树：？
          • 树：?
            P.S. List 按进入顺序保存对象且允许多个 NULL 元素对象；但 Set 只允许一个NULL;且两者都可以使用 Iterator 取出所有元素遍历，但 List 也可以使用 get(int index) 获取指定下表的元素
            P.S. 这里的 Linked- 前缀代表 “链”
    • Queue 有序，可重复，拥有特定的排队规则
      • PriorityQueue Object[] 数组 来实现二叉堆
        • 二叉堆：？
      • ArrayQueue Object[] 数组 + 双指针

  • Map

    • Map 使用键值对(k-v)存储
      • HashMap 数组 + 链表
        • 进一步了解：哈希冲突，“拉链法”：？
      • HashTable 数组 + 链表
        • 数组是 HashTable 的主体，链表用来解决哈希冲突
      • TreeMap 红黑树
      • LinkedHashMap extends HashMap
        • 在 HashMap 结构基础上，增加了一条双向链表
          • 为什么？

问题：在了解集合的基本特性后，如何选用？

  • 特点
    • 存放元素值
      • 需要保持元素唯一性
        • Set 下的 TreeSet 或 HashSet
      • 不需要保持唯一性
        • List 下的 AList 或 LList
    • 存放键值对k-v
      • 需要排序？
        • TreeMap 可以排序，否则使用 HashMap
      • 需要保证线程安全？
        • ConcurrentHashMap
