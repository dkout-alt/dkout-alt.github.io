我们采用新的叙述方式：一切从问题出发，最终回归于定义（源码）

问题：首先，为什么要使用集合？

  • 假设情况：我需要存储一组类型相同（Optional）的数据
    • 使用容器 “数组”
      • 数组的长度声明后不可变，怎么处理？
      • 如果类型不再相同，数组也无法储存。
      • 数组存储的是有序，可重复的对象。如果要存储的对象并非有序可重复呢？
    • 使用集合，能存储不同类型，不同数量的对象，可以保存具有映射关系的数据

问题：说说 List, Set, Queue, Map 的区别

“是什么” 与 底层数据结构

  • List, Set, Queue 都继承于 Collection 且实现了 java.lang.Iterable 接口

    • List 有序，可重复 <!– HyperLink to “深入了解 List” >

      • HashSet 基于 HashMap 实现的；无序，唯一
      • LinkedHashSet 是 HashSet 的子类，内部通过 LinkedHashMap 实现
      • TreeSet 红黑树（自平衡的排序二叉树）；有序，唯一
        • 红黑树：？
          • 树：?

    • Queue 有序，可重复，拥有特定的排队规则
      • PriorityQueue Object[] 数组 来实现二叉堆
        • 二叉堆：？
      • ArrayQueue Object[] 数组 + 双指针

  • Map

    • Map 使用键值对(k-v)存储

  • 特点
    • 存放元素值
      • 需要保持元素唯一性
        • Set 下的 TreeSet 或 HashSet

      • 需要排序？
        • TreeMap 可以排序，否则使用 HashMap

        • ConcurrentHashMap

一些 Java SE 的知识补全.md

continue, break and return

  • continue
    • 跳出当前循环，继续下一次循环
  • break
    • 跳出循环体

静态方法，静态变量 static

  • 静态方法只允许访问本类的静态成员，不允许访问实例成员
  • 静态方法也不支持使用 this ，因为this无法指向任何实例

重载与重写

  • 重写：当继承时,子类的方法要做出有别于父类的响应
    • 无法重写：被 private/final/static 修饰的方法和构造方法
    • 抛出异常：更小或相等
    • 两同（方法名和形参列表）两小（返回值和异常,不过只有引用类型可以返回子类，否则要相等）一大(访问权限)
  • 重载：一个方法，根据输入数据的不同，处理不同。如

基本类型和包装类型（？）

  • 包装类型的常量池技术
    • 所有整型包装类对象之间的比较，全部使用 equals 方法
    • 记住这个例子
  • 自动拆装箱
    • 自动装箱：将基本类型用对应的引用类型包装起来
    • 拆箱：将包装类型转换为基本数据类型

面向过程，面向对象

  • 面向过程将解决问题的过程拆成一个个方法，通过一个个方法的执行解决问题；而后者是 抽象对象 => 执行方法
  • 面向对象
    • 三大特征：封装，继承，多态
      • 多态：父类的引用指向子类的实例

成员变量与局部变量

  • 局部变量位于栈定义在方法的内部，成员变量位于堆在方法外部，直接写在类当中

对象实体，对象引用

  • new 创建对象实例，实例存在于堆内存中。对象引用指向对象实例，存放在栈内存中。
  • 构造方法：初始化
  • 构造方法不能被重写，但可以被重载，你可以在一个类中见到多种构造方法

接口和抽象类abstract

  • 后者经常用于代码复用，强调的是所属关系
  • 后者作为一个类，只能继承一个类但是可以实现多个接口
  • 注意：接口中的成员变量只能是 psf 类型的( public static final )

深拷贝与浅拷贝

  • 前者完全复制整个对象，包括其内部对象
  • 后者在堆上创建一个新对象，但是源对象内部的属性是引用类型的话，浅拷贝只复制内部对象的引用地址，也就是说他们公用同一个内部对象
  • 引用拷贝：两个不同的引用指向同一个对象

hashcode() 的作用

  • 它是干什么的？返回 int 的散列码，确定该对象在hash表中的索引位置
    • 谈到 hashcode() 时，我们要看 HashSet 的运作方式
      • 在一个对象被加入 HashSet 时，hashcode 被用来判断加入的位置以及查重。若code重复，则再次调用equals确定这俩对象是否真的相同
      • 如果真的相同，不可加入。但如果不相同，就会重新散列其他位置。
    • 同时提供 hashcode 和 equals 有利于快速判断某元素是否已经存在于对应容器。

String, StringBuffer, StringBuilder

  • 可变，线程安全，性能
    • 后两者继承自 AbstractStringBuilder 类，在其中用字符数组保存字符串。
      • StringBuffer 加了同步锁，StringB 没有加
      • SBd 更快，但是牺牲了线程安全性，多线程大量数据用 StringBuffer 比较靠谱

字符串常量池

  • 编译时可以确定内容的字符串，JVM将其存入字符串常量池

(伪)泛型，类型擦除与通配符

  • 为什么说是伪泛型？
    • 运行期间，所有的泛型信息都会被擦掉 ==> 类型擦擦除
    • 示例如下，通过反射判断泛型信息是否会被擦掉
  • 使用方式：泛型类，泛型接口，泛型方法

异常

  • Throwable 常用方法
    • String getMessage 返回异常发生时的简要描述
    • String toString 返回异常发生时的详细信息
    • String getLocalizedMessage 获得异常对象的本地化信息
    • void printStackTrace 打印 Throwable 对象封装的异常信息

(反)序列化：数据结构或对象 <==> 二进制字节流

  • 不想序列化的变量：使用 transient 关键字修饰, 只能修饰变量
  • static 变量不属于任何对象，有没有这个关键字修饰都不会被序列化

SQL Join: 从多个表获取结果
    INNER JOIN 在表中存在至少一个匹配时，该关键字返回行，如
        SELECT COLUMN_NAME FROM TABLE_1
        INNER JOIN TABLE_2
        ON table1.columnname=table2.columnname;
    LEFT JOIN 会从左表中返回所有的行，即使右表中没有匹配的行。“穷人”
    RIGHT JOIN 相反，"赔钱货"
    FULL JOIN 只要其中某个表存在匹配，就返回行，如
        现在，我们希望列出所有的人，以及他们的定单，以及所有的定单，以及定购它们的人。
        SELECT Persons.LastName, Persons.FirstName, Orders.OrderNo
        FROM Persons
        FULL JOIN Orders
        ON Persons.Id_P=Orders.Id_P
        ORDER BY Persons.LastName

UNION 用来合并两个或多个 SELECT 语句的结果集，两个结果集列数量必须相同，数据类型必须相似， 每条S语句中的列的顺序必须相同
    它默认选取不同的值，UNION ALL 将允许重复的值,如 列出中美的所有雇员
        SELECT E_Name FROM Employees_China
        UNION ALL
        SELECT E_Name FROM Employees_US
